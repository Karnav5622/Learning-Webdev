## 1. HTML DOCTYPE Declaration

### What is DOCTYPE?

The **DOCTYPE (Document Type Declaration)** is a special instruction placed at the very top of an HTML document, before the `<html>` tag. Its purpose is to tell the browser how to interpret and render the HTML code.

```html
<!DOCTYPE html>
```

### Key Points

* DOCTYPE is **not an HTML tag**.
* It is **not case-sensitive**.
* For all modern websites, **HTML5 DOCTYPE** should be used.
* It helps browsers decide **how to render** the page correctly.

### Rendering Modes

The DOCTYPE triggers a browser rendering mode:

* **Standards Mode**: Follows modern web standards (recommended).
* **Quirks Mode**: Uses legacy behavior for older websites (not recommended).

Using `<!DOCTYPE html>` ensures **Standards Mode**.

### Example (HTML5)

```html
<!DOCTYPE html>
<html>
<body>
    <p>Welcome to my website</p>
</body>
</html>
```

### What Happens Without DOCTYPE?

* Browser may switch to **Quirks Mode**
* Inconsistent layout and CSS behavior
* Cross-browser rendering issues
* Modern CSS (Flexbox, Grid) may break
* Unpredictable behavior during development

### DOCTYPEs for Different HTML Versions

| HTML Version           | DOCTYPE Declaration                                                                                             |
| ---------------------- | --------------------------------------------------------------------------------------------------------------- |
| HTML5                  | `<!DOCTYPE html>`                                                                                               |
| HTML 4.01 Strict       | `<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">`                    |
| HTML 4.01 Transitional | `<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">`        |
| XHTML 1.0 Strict       | `<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">` |
| XHTML 1.1              | `<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">`             |

### Best Practices

* Always include DOCTYPE
* Use HTML5 DOCTYPE for new projects
* Validate HTML using W3C Validator

## 2. HTML Basics

### What is HTML?

HTML (HyperText Markup Language) is the standard language used to **structure web pages**. It defines elements like text, images, links, tables, and multimedia.

HTML is used by **over 95% of websites**, making it the foundation of web development.

---

## 3. HTML Document Structure

### Basic Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>HTML</title>
</head>
<body>
    <p>GeeksforGeeks is an online study platform</p>
</body>
</html>
```

### Important Tags

| Tag       | Description                      |
| --------- | -------------------------------- |
| `<html>`  | Root element of the document     |
| `<head>`  | Metadata, title, links, SEO info |
| `<title>` | Page title shown in browser tab  |
| `<body>`  | Visible page content             |

---

## 4. HTML Headings

HTML provides six heading levels from `<h1>` to `<h6>`.

```html
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```

* `<h1>` is the most important
* Headings define **content hierarchy**

## 7. HTML Comments

Comments are ignored by browsers and used for documentation.

### Syntax

```html
<!-- Single-line comment -->

<!--
Multi-line
comment
-->
```

### Best Practices

* Explain *why*, not *what*
* Keep comments updated
* Avoid unnecessary comments

## 8. HTML Images

The `<img>` tag embeds images.

```html
<img src="image.png" alt="Description">
```

* `src`: image source
* `alt`: alternative text (important for accessibility)

## 10. HTML Tables

### Basic Table Structure

```html
<table>
  <tr>
    <th>Firstname</th>
    <th>Lastname</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>Priya</td>
    <td>Sharma</td>
    <td>24</td>
  </tr>
</table>
```

### Table Tags

| Tag          | Description     |
| ------------ | --------------- |
| `<table>`    | Table container |
| `<tr>`       | Table row       |
| `<th>`       | Header cell     |
| `<td>`       | Data cell       |
| `<caption>`  | Table title     |
| `<thead>`    | Header section  |
| `<tbody>`    | Body section    |
| `<tfoot>`    | Footer section  |
| `<colgroup>` | Column grouping |

---

## 11. Styling Tables with CSS

### Borders

```css
table, th, td {
  border: 1px solid black;
}
```

### Collapsed Borders

```css
table {
  border-collapse: collapse;
}
```

### Cell Padding

```css
th, td {
  padding: 20px;
}
```

### Text Alignment

```css
th {
  text-align: left;
}
```

### Border Spacing

```css
table {
  border-spacing: 5px;
}
```

## 15. Nested Tables

A table inside another table.

```html
<table>
  <tr>
    <td>
      <table>
        <tr><td>Inner Table</td></tr>
      </table>
    </td>
  </tr>
</table>
```

> Note: Nested tables can make layouts complex and are generally avoided in modern web design in favor of CSS layout systems.
